# 内存模型

这部分主要讨论变量的存储持续性、作用域和链接性。特别是静态变量的存储持续性和链接性。

在 C++11 中有 4 种变量存储的持续性方式。

  • 自动存储持续性，在函数定义中声明的变量，包括使用 auto 和 register 定义的变量
  • 静态存储持续性，在函数外定义的变量和使用 static 定义的变量，在程序运行整个过程中存在，包括使用 static const 定义的变量
  • 动态存储持续性，如果使用 thread_local 定义的变量，其生命周期和线程一样长
  • 线程存储持续性，使用 new 等分配在堆上的变量，直到 delete 进行释放

作用域描述的是名称在单元内的可见范围。

  • 局部，自动变量，即在代码块中的变量
  • 全局，单元内可见，一般指静态变量，但是静态变量也可能是局部的
  • 函数原型，只在包含参数列表的括号内可用
  • 类，类中申明的变量
  • 名称空间，存在于整个命名空间

链接性描述的是名称在单元间的可见范围。

  • 外部链接性，名称可在单元间共享
  • 内部链接性，名称只能在一个单元内共享，自动变量没有外部链接性
  • 无链接性，名称不能共享

函数的作用域可以是类或者名称空间，不能是局部的，所以不能在代码块中定义函数。

# 自动变量

在C++11 前使用 auto 进行定义。还有一种是寄存器变量，在 C++11 前，使用 register 进行定义，但在 C++17 中移除了该定义。自动变量作用域局部，且是无链接性的，使用栈进行管理。

# 静态变量

静态变量有三种链接性

  • 外部链接性，可在其他文件中访问
  • 内部链接性，只能在当前文件中访问
  • 无链接性，只能在代码块中访问

静态变量的作用域可能是局部，或者是全局的。

int global {100};				// 全局作用域，外部链接
static int one_file {100};		// 全局作用域，内部链接
int main()
{
    static int count {0};		// 局部作用域，无链接
}

比较麻烦的是，外部链接说明该变量在所有单元中都可见，但是变量只能定义一次，其他地方引用需要使用 extern 关键字。

// file1.cpp
int global {100};		// 定义声明

// file2.cpp
extern global;			// 引用声明，且不可以再次初始化

# 存储说明和 cv 限定

volatile 是为了改善编译器的优化能力，防止编译优化。

mutable 是为了指出即使结构的类型是常量，某个成员仍然可以被修改。

struct
{
    char name[30];
    mutable double access;
    int len;
} const veep{"123456", 0.1564156, 6};

veep.access += 3;			// 可以修改
veep.len += 2;				// 不可修改

关于 const 的几点说明

  • const 对默认存储类型有影响，默认情况下全局变量是外部链接的，但 const 全局变量是内部链接的。
  • 当在头文件中定义常量时，每个被引用的文件都有一个内部的常量，数据不共享。
  • 强制定义外部链接的常量，需要使用 extern const int count {100}

# 函数与语言的链接性

函数的存储性是静态的，也就是说函数存在在代码段，不在栈和堆中。函数是外部链接性的，所有单元共享，需要使用 extern 来引入定义的函数；也可以使用 static 来声明内部链接的函数。

static int private(double x);

static int private(double x)
{
	...
}

# 动态变量

用堆管理。需要注意的是， new 还有一种变体，被称为定位运算符。可以指定分配内存的地址。

char buf[500];
int* p = new(buf) int[20];		// 指定用 buf 来分配
cout << (int*)buf << endl;
cout << &(*p) << endl;

  • 程序不跟踪哪些内存单元被使用，内存管理被交给程序员。
  • 无法删除内存空间，因为整个空间是静态分配或者是在栈上的，使用 delete 会引发错误。

命名空间是全局的，也可以嵌套，但不能在代码块中，默认情况下，是外部链接性的。还有一个全局名称空间，对应文件声明区域，所以全局变量都是在全局名称空间中。

using 声明使特定的标识符可用， using 编译指令使整个名称空间可用。

using Jill::pail;
double pail;			// 错误

using namespace Jill;
double pail;			// 覆盖 Jill::pail
