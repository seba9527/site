配置树莓派4B

# 参数

  • Broadcom BCM2711，四核Cortex-A72（ARM v8）64位SoC @ 1.5GHz
  • 1GB，2GB或4GB LPDDR4-3200 SDRAM（取决于型号）
  • 2.4 GHz和5.0 GHz IEEE 802.11ac无线，蓝牙5.0，BLE
  • 千兆以太网
  • 2个USB 3.0端口；2个USB 2.0端口。
  • Raspberry Pi标准40针GPIO接头连接器（与以前的板完全向后兼容）
  • 2个micro-HDMI端口（最多支持4kp60）
  • 2通道MIPI DSI显示端口
  • 2通道MIPI CSI摄像机端口
  • 4针立体声音频和复合视频端口
  • H.265（4kp60解码），H264（1080p60解码，1080p30编码）
  • OpenGL ES 3.0图形
  • Micro-SD卡插槽，用于加载操作系统和数据存储
  • 通过USB-C连接器5V DC（最小3A *）
  • 通过GPIO接头提供5V DC（最小3A *）
  • 启用以太网供电（PoE）（需要单独的PoE HAT）
  • 工作温度：0 – 50摄氏度环境

详细参数

# SD卡

首先配置SD卡，格式化为FAT32的，不能是EXFAT的，貌似不支持，参见参考文献。然后去官网下镜像，当然我这里是去清华TUNA下的，很快，几秒就下完了。然后用Etcher烧录镜像。

# 开启SSH

烧录完成后，打开SD卡目录，直接在boot盘中中添加ssh文件，为空就行了。 默认用户名pi，密码raspberry。

如果完成上面的操作依然提示"connection refused"，则说明 SSH 服务没有启动。这时需要在树莓派命令行下手动启用 SSH 服务，命令如下：

service sshd restart
1

因为我没有键盘也没有显示器，只能通过ssh连上去，同时还需要配置wifi，不然连ip都找不到，

country=CN

update_config=1
network={
  ssid="WiFi名字，不删除引号,不能有中文"
  psk="WiFi密码，不删除引号"
  priority="替换为数字，这里删除引号，数字越大优先级越高"
}

# 5G wifi

貌似连不上我自己路由器的5G信号，查了一下，有点说法的，参见参考文献。

使用iwlist channel命令可以查看到当前允许连接的频段。因为将地区设置为CN后，5G可以使用的频段仅有149-165一共5个。这个范围其实是远小于中国实际可以使用的信道的。早在2012年，中国就新开放了36-64频段，因此实际能使用的频段至少有36, 40, 44, 48, 52, 56, 60, 64, 149,153, 157, 161, 165，一共13个。

# docker ipv6

在docker内部启动ipv6，需要配置。有2种方法。第一种是设置ipv6这个选项。

sudo vim /etc/docker/daemon.json
{
  "ipv6": true,
  "fixed-cidr-v6": "2001:db8:1::/64"
}
1
2
3
4
5

然后重写加载docker配置sudo systemctl daemon-reload，并重启docker，之后把portainer装起来。

后续run的时候加入参数--ip6，使用默认网关管理，进行dhcpv6。这时候还是不能ping通外网的，内部的网络已经可以向外网通信，但外网无法得知内网的路由。Linux 提供了一个 proxy_ndp 选项，可以让外网的ndp请求穿过网关。在网关上敲

# 设置ndp协议
sudo sysctl net.ipv6.conf.all.proxy_ndp=1
sudo ip -6 neigh add proxy 240e:3a1:54e2:f120::2 dev eth0
1
2
3

对于局域网中每一个IPv6地址，都要运行一遍这个命令，才能使得整个局域网顺利接入IPv6网络, 非常繁杂。有人说可以用ndp6，但我没有试。

同样都有的缺点是IPv6无法像IPv4一样仅暴漏端口，IPv6下地址全端口都开放。 参考

而且这里需要每次重启都重新设置sudo sysctl net.ipv6.conf.all.proxy_ndp=1。

# netdata

netdata是个非常强的监控软件，我这边还是用docker安装。

# 创建网关
docker network create --driver bridge IPv6Net \
  --ipv6 \
  --subnet "240e:3a1:54e2:f120::/80" \
  --gateway="240e:3a1:54e2:f120::1" \
  --subnet=172.30.0.0/16 \
  --gateway=172.30.0.1

# 开启服务
docker run -d \
  --name=netdata \
  --network=IPv6Net \
  --ip=172.30.0.2 \
  --ip6="240e:3a1:54e2:f120::2" \
  -p 19999:19999 \
  --restart=always \
  -v /etc/passwd:/host/etc/passwd:ro \
  -v /etc/group:/host/etc/group:ro \
  -v /proc:/host/proc:ro \
  -v /sys:/host/sys:ro \
  -v /var/run/docker.sock:/var/run/docker.sock:ro \
  --cap-add SYS_PTRACE \
  --security-opt apparmor=unconfined \
  netdata/netdata

sudo ip -6 neigh add proxy 240e:3a1:54e2:f120::2 dev eth0

# 显示温度

默认好像没有温度，通过文档，可以在这里collectors，找到相关说明，我们选择sensor，然后按照他的配置修改即可。

cd /etc/netdata   
# Replace this path with your Netdata config directory, if different
./edit-config charts.d/sensors.conf


#This is the internal default for charts.d/sensors.conf

# the directory the kernel keeps sensor data
sensors_sys_dir="${NETDATA_HOST_PREFIX}/sys/devices"

# how deep in the tree to check for sensor data
sensors_sys_depth=10

# if set to 1, the script will overwrite internal
# script functions with code generated ones
# leave to 1, is faster
sensors_source_update=1

# how frequently to collect sensor data
# the default is to collect it at every iteration of charts.d
sensors_update_every=

# array of sensors which are excluded
# the default is to include all
sensors_excluded=()

# Samba

SMB（Server Message Block）通信协议是微软和英特尔在1987年制定的协议，主要是作为Microsoft网络的通讯协议，其不仅提供目录和打印机共享，还支持认证、权限设置。本文是树莓派上安装samba的教程，使得树莓派在Windows和Mac等系统间的文件共享变得方便，同时，由于现在的电视基本上都支持Samba协议，所以树莓派安装Samba后，就可以在电视上直接看存储在树莓派上的电影啦。

安装的话直接用apt安装就行了。

samba的配置文件是/etc/samba/smb.conf，要配置Samba，就需要修改这个配置文件。修改配置如下

[pi]
# 共享名称为
   comment = zjgcjy raspberry
   path = /pi
   browseable  =  yes
   writable  =  yes
   # 任何人都具有了访问修改的权限
   # 因为是公共文件夹，所以给了所有用户全部权限，可以自定义
   create mask = 0664
   # 新创建文件的默认属性
   directory mask = 0775
   #新创建文件夹的默认属性
   write list = pi
   valid users = pi

# 挂载硬盘

最后就是挂载USB移动硬盘了。这边介绍几个命令。首先，如果是ntfs或者exfat，需要先安装几个包，不然不能写入，最新的pi4好像是自带了，NTFS-3g是一个开源软件，它支持在Linux下面读写NTFS格式的分区。它非常的快速，同时也很安全。用lsmod能看到fuse模块已经加载了。

# 安装所需软件包
sudo aptitude install ntfs-3g exfat-fuse
# 加载内核模块
modprobe fuse
